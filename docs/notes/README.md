# 专题笔记

本目录保存已经学习、适合复习的后端知识。每份笔记只聚焦一个主题，建议使用有序的 kebab-case 文件名，例如：

```text
01-request-lifecycle.md
02-dependency-injection.md
03-dto-validation.md
```

专题笔记建议包含：

1. 这个概念解决什么问题。
2. 与前端经验的类比及类比的边界。
3. 它在当前项目中的实际位置。
4. 最小示例和常见错误。
5. 用于确认理解的复习问题。

不要提前批量创建尚未学习的笔记。完成课程并形成稳定理解后再新增；如果理解发生变化，应修正笔记，并在学习日志中记录变化。
