# 后端学习路线

路线采用两轮制。第一轮优先在 30 天内形成可运行、可测试、可部署的全栈闭环；第二轮再深化复杂场景。状态只有在学习者能够解释并在当前项目中识别或使用相关知识后才提升，不以代码由谁输入作为唯一标准。

## 第一轮｜30 天 AI 辅助全栈闭环（进行中）

详细日期、里程碑和完成标准见 `docs/30-day-fullstack-loop.md`。

### 第 1 周｜NestJS、CRUD 与 PostgreSQL（已完成）

- 请求与响应生命周期
- Module、Controller、Service、依赖注入
- DTO、Pipe、异常和状态码
- 完成进程内 Projects CRUD（已完成）
- 连接 PostgreSQL，选择 ORM 并运行迁移（已完成）
- 将 Projects CRUD 从内存迁移到真实数据库（已完成）

### 第 2 周｜数据边界、认证与授权（进行中）

- 用户表、项目归属、约束和基础索引
- 注册、登录和密码哈希
- 选择并实现 Cookie、Session 或 JWT 方案
- Guard、当前用户和基础 RBAC
- 用户只能访问和修改自己的项目

### 第 3 周｜前后端联调与本地工程化（未开始）

- 前端接入真实 API 和认证流程
- 统一错误响应、配置、环境变量和 OpenAPI
- 关键单元测试与 e2e
- Dockerfile 与 Compose 运行 API 和 PostgreSQL
- 使用日志定位常见请求和数据库错误

### 第 4 周｜部署与闭环验收（未开始）

- 部署 API、数据库和前端
- 生产环境变量、迁移、CORS、Cookie 与 HTTPS
- 验证注册、登录、权限和项目 CRUD
- 根据线上日志定位并修复至少一个问题
- 完成代码、安全、数据与部署清单审查

## 第二轮｜后端能力深化（未开始）

- PostgreSQL 关系、事务、并发、索引和性能
- Repository 边界、集成测试与更完整的测试策略
- Redis、缓存、限流、队列和分布式锁
- 文件处理、WebSocket、异步任务和第三方服务
- 日志、监控、CI、回滚和生产可靠性
- 按需增加实时通信、异步任务等独立学习项目

工程组织和 Git 学习贯穿两轮，不再作为阻塞闭环进度的独立阶段。路线可根据实际进度调整，但第一轮必须保持端到端交付方向。
