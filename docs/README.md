# 学习档案说明

`docs/` 用于保存学习目标、当前进度、历史过程和已经形成的知识。它让项目在被 Git clone 到其他设备后，新 Agent 不依赖旧对话也能继续担任后端导师。

## 文档结构

| 路径 | 职责 | 更新时机 |
| --- | --- | --- |
| `learning-progress.md` | 当前学习状态的唯一快照 | 每次课程或练习结束后 |
| `roadmap.md` | 后端学习路线和阶段状态 | 阶段开始、完成或调整时 |
| `learning-log.md` | 按时间追加的学习记录 | 每次发生有效学习后 |
| `decisions.md` | 长期有效的教学与技术决策 | 作出重要取舍时 |
| `notes/` | 按主题整理的可复习知识 | 形成可复用知识时 |

## 新会话读取顺序

先阅读 `learning-progress.md`，再查看 `learning-log.md` 的最新一条记录和 `roadmap.md` 的当前阶段。只有当前任务相关时才继续读取专题笔记或决策记录。最后检查实际代码和 Git 状态，避免仅凭文档假设项目现状。

## 维护规则

- `learning-progress.md` 只描述当前事实，过期内容应被更新。
- `learning-log.md` 只追加记录，不修改旧结论；后续理解有变化时新增一条记录。
- 使用 `YYYY-MM-DD` 日期，并区分“接触过、理解中、可独立应用”。
- 不记录对恢复学习无帮助的聊天内容、工具过程或大段代码副本。
- 课程代码、测试和对应的文档更新应尽量进入同一次提交。
- 一个学习阶段形成完整、可验证的结果后，由 Agent 主动创建本地 Git 提交；推送远端不在该授权范围内。
